## 🏗 מבנה האפליקציה

האפליקציה בנויה ממספר רכיבים מרכזיים:

- **Frontend (צד לקוח):**
  - **Next.js:** מסגרת העבודה (Framework) המובילה ל-React (גרסה 14+ עם App Router).
  - **Tailwind CSS:** ספריית עיצוב לפיתוח מהיר.
  - **ShadCN UI:** אוסף רכיבי UI מודרניים ונגישים הממוקמים בתיקיית `components/ui`.
  - **RTL Support:** מוגדר ברמת ה-Layout (`dir="rtl"`) לתמיכה מלאה בעברית.

- **Backend (צד שרת):**
  - **Convex:** פלטפורמת Backend-as-a-Service המספקת מסד נתונים בזמן אמת, פונקציות שרת (Server Functions) ואימות משתמשים.
  - **Convex Auth:** מערכת אימות משתמשים מאובטחת המוטמעת ישירות ב-Convex (תומכת ב-Next.js בצורה חלקה).

### תיקיות חשובות:
- `app/`: מכילה את דפי האתר והניווט (Next.js App Router).
  - `(auth)/`: דפי התחברות והרשמה.
  - `checkout/`: נתיב API ל-Polar Checkout.
  - `success/`: דף הצלחה לאחר תשלום.
  - `globals.css`: הגדרות עיצוב גלובליות.
- `convex/`: מכילה את לוגיקת השרת (Schema, פונקציות, הגדרות אימות).
- `components/`: רכיבי UI ורכיבים לשימוש חוזר.
  - `payments/`: רכיבי תשלום (PaywallModal, PremiumGate).
- `config/`: קבצי קונפיגורציה מרכזיים.
  - `appConfig.ts`: דגלי תכונות וקונפיגורציה כללית.
  - `polarConfig.ts`: הגדרות Polar (תשלומים).
  - `planDisplay.ts`: טקסטים ותמחור לתוכניות.
- `lib/`: ספריות עזר (כגון `utils.ts`).

## 🚀 צעדים ראשונים (עם קבלת התבנית)

> 📖 **מדריך מפורט:** ראה `docs/setup.md` להוראות התקנה מלאות עם צילומי מסך.

### דרישות מקדימות

לפני שמתחילים, התקינו את הכלים הבאים:

#### 1. Node.js

**🪟 Windows:**
1. גשו ל-[nodejs.org](https://nodejs.org) והורידו את גרסת LTS
2. בזמן ההתקנה, וודאו שהאפשרות **"Add to PATH"** מסומנת

**🍎 Mac:**
1. גשו ל-[nodejs.org](https://nodejs.org) והורידו את גרסת LTS (קובץ `.pkg`)
2. או דרך Homebrew: `brew install node`

#### 2. Git

**🪟 Windows:**
1. גשו ל-[git-scm.com/download/win](https://git-scm.com/download/win)
2. הורידו והתקינו **Git for Windows**

**🍎 Mac:**
1. פתחו Terminal והריצו: `git --version`
2. אם לא מותקן, Mac יציע להתקין Xcode Command Line Tools - לחצו Install
3. או דרך Homebrew: `brew install git`

#### 3. Bun

לאחר התקנת Node.js, הריצו בטרמינל:
```bash
npm i -g bun
```

#### 4. אימות ההתקנות

```bash
node -v
git --version
bun -v
```

> ❌ **אם `node -v` לא עובד ב-Windows:** סגרו את Cursor/VSCode לחלוטין ופתחו מחדש.

### הורדת התבנית

1. הורידו את התבנית מהפלטפורמה
2. חלצו את ה-ZIP לשולחן העבודה
3. פתחו את התיקייה ב-Cursor/VSCode

## 🗄 הגדרת מסד נתונים (Convex)

כדי שהאפליקציה תעבוד, עליכם לקשר אותה לפרויקט Convex משלכם:

1. **יצירת חשבון Convex:** הירשמו ב-[convex.dev](https://convex.dev).
2. **התחברות דרך הטרמינל:**
   הריצו את הפקודה בתיקיית הפרויקט:
   ```bash
   bunx convex login
   ```
3. **יצירת פרויקט חדש:**
   הריצו את הפקודה ליצירת פרויקט וקישורו:
   ```bash
   bunx convex dev
   ```
   פקודה זו תבקש מכם לבחור שם לפרויקט וליצור אותו. בסיום, היא תיצור קובץ `.env.local` עם כתובת השרת שלכם (`CONVEX_DEPLOYMENT` ו-`NEXT_PUBLIC_CONVEX_URL`).

## 📦 התקנה והרצה

### 1. התקנת חבילות
התקינו את כל התלויות של הפרויקט באמצעות bun:
```bash
bun install
```

### 2. הרצת השרת (Convex)
פתחו טרמינל נפרד והריצו את שרת הפיתוח של Convex (כדי לסנכרן שינויים ב-Backend בזמן אמת):
```bash
bunx convex dev
```
(השאירו את הטרמינל הזה פתוח ברקע).

### 3. הרצת האפליקציה (Next.js)
בטרמינל נוסף, הריצו את שרת הפיתוח של האתר:
```bash
bun run dev
```
פתחו את הדפדפן בכתובת [http://localhost:3000](http://localhost:3000) כדי לראות את האפליקציה.

## 🎯 פיצ'רים מרכזיים

### מערכת תשלומים (Polar)
האפליקציה כוללת אינטגרציה מלאה עם [Polar](https://polar.sh) למערכת תשלומים:
- **Paywall Modal:** מודל תשלום בעברית עם 3 תוכניות (חינם, חודשי, שנתי)
- **PremiumGate:** רכיב מעטפת שמגן על עמודים ומציג Paywall למשתמשים חינמיים
- **Mock Payments:** מצב בדיקה שמאפשר לבדוק את ה-Paywall בלי תשלום אמיתי

📖 **מדריך הגדרה:** ראה `docs/POLAR_PAYMENTS_SETUP.md` להגדרה מפורטת.

### קונפיגורציה מרכזית (`appConfig.ts`)
קובץ קונפיגורציה מרכזי לניהול דגלי תכונות:
- `PAYWALL_ENABLED`: הפעלה/כיבוי של ה-Paywall
- `PAYMENT_SYSTEM_ENABLED`: הפעלה/כיבוי של מערכת התשלומים האמיתית
- `MOCK_PAYMENTS`: מצב בדיקה לתשלומים מדומים
- `IS_DEV_MODE`: זיהוי אוטומטי של מצב פיתוח

### קונסולת דיבאג (Dev בלבד)
במצב פיתוח, כפתור באג צהוב מופיע ב-Navbar (פינה שמאלית עליונה) ומאפשר:
- הצגת מצב האפליקציה (Paywall, תשלומים, סטטוס משתמש)
- תצוגה מקדימה של מודלים (התחברות, הרשמה, Paywall)
- שדרוג מהיר לפרימיום (במצב Mock)

### מחיקת חשבון
משתמשים יכולים למחוק את החשבון שלהם דרך תפריט המשתמש ב-Navbar, עם אישור דו-שלבי.

## 💡 טיפים נוספים

- **עברית (RTL):** האפליקציה מוגדרת לעבוד מימין לשמאל באמצעות תגית ה-`html` ב-`app/layout.tsx`. כל רכיב חדש שתצרו יירש את הכיווניות הזו.
- **אבטחה:** מפתחות API סודיים לא נשמרים בקוד אלא במשתני סביבה. ודאו שקובץ `.env.local` לא עולה ל-Git.
- **פריסה (Deployment):** הדרך הקלה ביותר לפרוס את האפליקציה היא דרך [Vercel](https://vercel.com), יוצרי Next.js.
- **תשלומים:** לפני פריסה לייצור, ודאו שמוגדרים כל משתני הסביבה הנדרשים (ראה `docs/POLAR_PAYMENTS_SETUP.md`).
